## Overview

One of the most critical challenges in this UI is helping users understand the relationships between entities. The shared library pattern means:

- Editing an Option affects all MSIs that link to it
- Deleting an UpCharge removes it from multiple MSIs
- Pricing changes propagate to all linked items

This document details how we visualize these relationships and communicate impact.

---

## Design Principles

1. **Always show context** - When viewing an entity, always show where it's used
2. **Warn before impact** - Any action affecting multiple items shows clear warning
3. **Bi-directional navigation** - Easy to go from MSI â†’ Option â†’ MSI
4. **Visual consistency** - Same patterns for Options, UpCharges, Additional Details

---

## "Where Used" Panel

Every shared entity (Option, UpCharge, Additional Detail) has a "Where Used" panel.

### Location

- **In Library views:** Right sidebar when item is selected
- **In Detail views:** Dedicated section or tab
- **In Edit modals:** Warning banner at top
